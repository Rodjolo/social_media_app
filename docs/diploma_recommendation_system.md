# Система рекомендаций для защиты диплома

## Что реализовано

В мобильное Flutter-приложение встроен отдельный модуль рекомендаций фильмов.

Пользователь может:

- открыть каталог фильмов;
- поставить фильму оценку;
- отметить фильм как понравившийся;
- получить персональные рекомендации.

## Как распределена логика

Мобильное приложение отвечает за:

- интерфейс;
- сбор оценок;
- показ рекомендаций.

Python-часть отвечает за:

- экспорт оценок из PocketBase;
- объединение их с MovieLens;
- построение рекомендаций;
- запись результата обратно в PocketBase.

## Какой алгоритм используется

Используется **item-based collaborative filtering**.

Упрощенно это работает так:

1. Строится матрица пользователь–фильм на основе оценок MovieLens.
2. В эту матрицу добавляется текущий пользователь приложения.
3. Для фильмов считается косинусное сходство.
4. Если пользователь высоко оценил один фильм, система ищет другие фильмы с похожим паттерном оценок.
5. Непросмотренные фильмы получают recommendation score и ранжируются.

Почему этот подход подходит для диплома:

- он академически узнаваем;
- его легко объяснить;
- он достаточно простой для рабочего прототипа;
- он действительно дает персонализированные результаты.

## Почему выбран MovieLens

MovieLens выбран потому что:

- это открытый и хорошо известный датасет;
- он содержит большое число реальных пользовательских оценок;
- он подходит для экспериментов с collaborative filtering;
- он избавляет от необходимости вручную придумывать искусственную обучающую базу пользователей.

В дипломе MovieLens можно описывать как историческую базу оценок системы, а текущего пользователя приложения — как нового пользователя, предпочтения которого анализируются на этом фоне.

## Что показать на защите

Удобный сценарий демонстрации:

1. Пользователь входит в приложение.
2. Пользователь оценивает 15–20 фильмов.
3. Администратор открывает панель рекомендаций.
4. Запускается пересчет рекомендаций.
5. Экран рекомендаций обновляется.
6. Показывается новый персонализированный список с постерами, жанрами, описаниями и score.

## Какие метрики можно озвучить

Для диплома достаточно даже простых метрик:

- сколько фильмов оценил пользователь;
- сколько рекомендаций построено;
- средний score рекомендаций;
- доминирующие жанры в выдаче;
- насколько изменилась подборка после новых оценок.

Эти данные уже можно использовать в админ-панели и в сравнительном отчете между запусками.

## Ограничения текущего решения

- Пересчет выполняется на компьютере, а не на самом телефоне.
- Алгоритм специально сделан простым и понятным для диплома, а не для промышленного продакшена.
- Качество рекомендаций зависит от количества оцененных фильмов.

## Практический результат

На текущем этапе система работает end-to-end:

- Flutter собирает оценки пользователя;
- PocketBase хранит пользователей, фильмы, оценки и рекомендации;
- Python вычисляет персональные рекомендации по MovieLens;
- приложение показывает результат пользователю.
